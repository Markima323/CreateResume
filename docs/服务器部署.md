# Hetzner 自动部署说明

本项目参考 `ZeitPlan` 的部署方式：每次向 GitHub `main` 分支提交后，GitHub Actions 先运行后端测试、前端构建和生产 Compose 校验；全部通过后，通过 SSH 登录 Hetzner，在服务器执行数据库备份、`git pull --ff-only` 和 `docker compose up -d --build`。

## 1. 部署架构

```text
Internet
  │ 80 / 443
  ▼
Nginx + Basic Auth + Let's Encrypt
  ├── /api/* → Spring Boot :8080
  └── /*      → React/Nginx :80

Internal Docker network
  ├── PostgreSQL
  └── persistent generation volume
```

PostgreSQL 和 Spring Boot 不开放公网端口。整个站点使用 Basic Auth，避免他人访问简历数据或消耗 OpenAI API 额度。

## 2. 首次准备服务器

以下命令以 Ubuntu 和 `/opt/create-resume` 为例。使用具有 `sudo` 权限的非 root 用户登录服务器：

```bash
sudo apt-get update
sudo apt-get install -y git ca-certificates curl
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker "$USER"
```

退出 SSH 并重新登录，使 Docker 用户组生效。确认：

```bash
docker version
docker compose version
```

配置防火墙前，确保 SSH 端口规则正确：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

## 3. DNS 与代码

在 DNS 服务商处创建域名的 A 记录，指向 Hetzner IPv4 地址。如果服务器配置了 IPv6，再创建 AAAA 记录。等待 DNS 生效后验证：

```bash
getent hosts resume.example.com
```

克隆项目：

```bash
sudo mkdir -p /opt/create-resume
sudo chown "$USER":"$USER" /opt/create-resume
git clone https://github.com/Markima323/CreateResume.git /opt/create-resume
cd /opt/create-resume
```

如果以后将仓库设为私有，需要为服务器配置只读 Deploy Key，保证 `git pull` 可执行。

## 4. 生产环境变量

```bash
cp deploy/.env.production.example .env
nano .env
chmod 600 .env
```

至少修改：

- `SERVER_NAME`：简历系统域名，不包含 `https://`。
- `CERTBOT_EMAIL`：Let's Encrypt 通知邮箱。
- `BASIC_AUTH_USER` 和 `BASIC_AUTH_PASSWORD`：访问网站时的登录信息。
- `POSTGRES_PASSWORD`：数据库随机长密码。数据库初始化后不要只修改此值；修改密码还必须同步执行 `ALTER USER`。
- `OPENAI_API_KEY`：OpenAI API Key，只进入后端容器。
- `OPENAI_MODEL`：账号可使用的模型 ID。

建议分别生成两个不同的随机密码：

```bash
openssl rand -base64 36
openssl rand -base64 36
```

如果密码包含特殊字符，请在 `.env` 中使用单引号包裹。

## 5. 首次签发 HTTPS 证书

确认域名已经指向服务器，而且端口 80 没有被其他程序占用：

```bash
chmod +x deploy/init-letsencrypt.sh
./deploy/init-letsencrypt.sh resume.example.com admin@example.com
```

把参数替换成 `.env` 中的真实域名和邮箱。证书创建后启动生产服务：

```bash
docker compose --env-file .env -f compose.prod.yml up -d --build
docker compose --env-file .env -f compose.prod.yml ps
```

首次构建会安装 LaTeX 依赖，因此可能需要几分钟。所有容器正常后，访问：

```text
https://resume.example.com
```

浏览器会先显示 Nginx Basic Auth 登录框。

## 6. GitHub Actions Secrets

在 GitHub 仓库打开：

```text
Settings → Secrets and variables → Actions → New repository secret
```

创建：

| Secret | 内容 |
|---|---|
| `HETZNER_HOST` | 服务器 IP 或 SSH 域名 |
| `HETZNER_USER` | 部署用户，例如 `deploy` |
| `HETZNER_DEPLOY_PATH` | `/opt/create-resume` |
| `HETZNER_SSH_KEY` | SSH 私钥的完整内容 |
| `HETZNER_SSH_PASSPHRASE` | 私钥密码，没有则不创建 |
| `HETZNER_PORT` | SSH 端口；使用 22 时可以不创建 |

建议在 `Settings → Environments` 创建 `production` Environment，并为生产部署开启审批保护。

服务器的公钥应存在于：

```text
~/.ssh/authorized_keys
```

可以在本机生成专用部署密钥：

```powershell
ssh-keygen -t ed25519 -C "github-create-resume-deploy" -f "$HOME\.ssh\create_resume_deploy"
```

- 将 `create_resume_deploy.pub` 内容加入服务器 `authorized_keys`。
- 将 `create_resume_deploy` 私钥完整内容保存到 `HETZNER_SSH_KEY`。

## 7. 自动部署流程

配置完成后，在本机提交并推送：

```powershell
git add .
git commit -m "Configure production deployment"
git push origin main
```

GitHub Actions 的 `Deploy to Hetzner` 工作流会：

1. 执行 Maven 测试。
2. 执行 React 生产构建。
3. 校验 `compose.prod.yml`。
4. SSH 登录服务器。
5. 如果数据库已运行，先创建 SQL 备份并保留 7 天。
6. 快进更新 `main` 分支。
7. 重新构建并启动容器。
8. 等待 backend 和 nginx 健康检查通过。
9. 清理七天前的无用 Docker 镜像。

也可以在 GitHub 的 Actions 页面使用 `Run workflow` 手工重新部署。

## 8. 运维命令

```bash
cd /opt/create-resume

# 状态
docker compose --env-file .env -f compose.prod.yml ps

# 日志
docker compose --env-file .env -f compose.prod.yml logs backend --tail 100
docker compose --env-file .env -f compose.prod.yml logs nginx --tail 100

# 手工部署
git pull --ff-only origin main
docker compose --env-file .env -f compose.prod.yml up -d --build

# 停止，不删除数据卷
docker compose --env-file .env -f compose.prod.yml down
```

不要在生产服务器执行 `docker compose down -v`，它会删除 PostgreSQL 和生成文件的数据卷。

## 9. 证书续期

`certbot` 容器每 12 小时检查一次证书，Nginx 每 6 小时自动 reload，以加载续期后的证书。检查证书日志：

```bash
docker compose --env-file .env -f compose.prod.yml logs certbot --tail 100
```

手工测试续期：

```bash
docker compose --env-file .env -f compose.prod.yml run --rm certbot renew --dry-run
```
