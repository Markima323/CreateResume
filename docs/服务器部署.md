# Hetzner 共享服务器部署说明

## 1. 最终域名与架构

- ZeitPlan：`https://zeitplan.markima.de`
- CreateResume：`https://lebenslauf.markima.de`

两个项目位于同一台 Hetzner 服务器。ZeitPlan 的 Nginx 是唯一公网入口并继续独占 `80/443`，根据域名转发到不同项目：

```text
zeitplan.markima.de ───┐
                      ├─ ZeitPlan Nginx :80/:443
lebenslauf.markima.de ─┘       │
                               ├─ ZeitPlan frontend/backend
                               └─ resume-frontend/resume-backend
```

两个 Compose 项目通过外部 Docker 网络 `public-proxy` 互通。CreateResume 的 PostgreSQL 只位于自己的内部网络，不会被 ZeitPlan 或公网访问。

Lebenslauf 域名额外启用 Nginx Basic Auth，防止公开访问简历数据或滥用 OpenAI API；ZeitPlan 的现有登录和访问方式不受影响。

## 2. 涉及的两个仓库

共享入口需要同时提交：

- `CreateResume`：生产服务加入 `public-proxy`，网络别名为 `resume-frontend` 和 `resume-backend`。
- `ZeitPlan`：Nginx 增加 `lebenslauf.markima.de` 虚拟主机、Basic Auth、证书和反向代理配置。

ZeitPlan 修改完成前，不要在服务器启动 CreateResume 的生产 Compose。

## 3. DNS

在 `markima.de` 的 DNS 服务商创建：

```text
类型：A
名称：lebenslauf
目标：Hetzner 服务器 IPv4
```

如果使用 IPv6，再创建对应 AAAA 记录。验证：

```bash
getent hosts lebenslauf.markima.de
```

返回的 IP 必须是当前 Hetzner 服务器。

## 4. 准备共享 Docker 网络

在服务器执行一次：

```bash
docker network inspect public-proxy >/dev/null 2>&1 || docker network create public-proxy
```

两个项目的 GitHub Actions 也会在网络缺失时自动创建。

## 5. 更新 ZeitPlan 服务器配置

进入服务器上的 ZeitPlan 目录，向 `.env` 增加：

```dotenv
RESUME_SERVER_NAME=lebenslauf.markima.de
PUBLIC_PROXY_NETWORK=public-proxy
```

不要修改现有 `SERVER_NAME=zeitplan.markima.de`。

为 Lebenslauf 创建访问账号。以下命令不会把密码写进 shell 历史：

```bash
cd /path/to/ZeitPlan
read -rsp "Lebenslauf password: " RESUME_PASSWORD
echo
docker run --rm httpd:2.4-alpine htpasswd -nbB lebenslauf "$RESUME_PASSWORD" > infra/nginx/.htpasswd
unset RESUME_PASSWORD
chmod 600 infra/nginx/.htpasswd
```

该文件已加入 `.gitignore`，只存在于服务器，不要提交。

提交并推送本机 `G:\Arbeit\ZeitPlan` 的共享代理改动后，ZeitPlan Actions 会自动更新服务器。也可以在服务器手工执行：

```bash
git pull --ff-only origin main
docker compose --env-file .env -f docker-compose.prod.yml up -d --build
```

## 6. 扩展现有 Let's Encrypt 证书

共享 Nginx 使用 ZeitPlan 已有证书 lineage，并将两个域名加入同一张证书。确认新 DNS 已生效，然后在 ZeitPlan 服务器目录执行：

```bash
docker compose --env-file .env -f docker-compose.prod.yml run --rm \
  --entrypoint certbot certbot certonly \
  --webroot -w /var/www/certbot \
  --cert-name zeitplan.markima.de \
  --expand \
  --non-interactive \
  --agree-tos \
  --email YOUR_EMAIL@example.com \
  -d zeitplan.markima.de \
  -d lebenslauf.markima.de
```

把邮箱替换为真实的 Let's Encrypt 联系邮箱。成功后重新加载 Nginx：

```bash
docker compose --env-file .env -f docker-compose.prod.yml exec nginx nginx -s reload
```

检查证书包含两个域名：

```bash
echo | openssl s_client -connect lebenslauf.markima.de:443 -servername lebenslauf.markima.de 2>/dev/null \
  | openssl x509 -noout -ext subjectAltName
```

## 7. 首次准备 CreateResume

在服务器创建独立目录：

```bash
sudo mkdir -p /opt/create-resume
sudo chown "$USER":"$USER" /opt/create-resume
git clone https://github.com/Markima323/CreateResume.git /opt/create-resume
cd /opt/create-resume
```

创建生产环境变量：

```bash
cp deploy/.env.production.example .env
nano .env
chmod 600 .env
```

示例：

```dotenv
PUBLIC_PROXY_NETWORK=public-proxy
POSTGRES_DB=resume
POSTGRES_USER=resume
POSTGRES_PASSWORD=使用随机长密码
OPENAI_API_KEY=sk-真实API-Key
OPENAI_MODEL=gpt-5.4-mini
```

数据库首次初始化后，如果只修改 `.env` 的数据库密码，已有 PostgreSQL 用户密码不会自动变化。

第一次手工启动：

```bash
docker network inspect public-proxy >/dev/null 2>&1 || docker network create public-proxy
docker compose --env-file .env -f compose.prod.yml up -d --build
docker compose --env-file .env -f compose.prod.yml ps
```

CreateResume 不绑定宿主机端口。正常情况下只显示容器内部的 `8080/tcp` 和 `80/tcp`，公网访问必须经过 ZeitPlan Nginx。

## 8. CreateResume GitHub Actions Secrets

进入 CreateResume GitHub 仓库：

```text
Settings → Secrets and variables → Actions
```

配置：

| Secret | 内容 |
|---|---|
| `HETZNER_HOST` | Hetzner IP 或 SSH 域名 |
| `HETZNER_USER` | SSH 部署用户 |
| `HETZNER_DEPLOY_PATH` | `/opt/create-resume` |
| `HETZNER_SSH_KEY` | SSH 私钥完整内容 |
| `HETZNER_SSH_PASSPHRASE` | 私钥密码，可选 |
| `HETZNER_PORT` | 非 22 端口时配置 |

建议创建 GitHub `production` Environment，并启用部署审批。

每次推送 CreateResume `main` 后，Actions 会：

1. 测试 Spring Boot。
2. 构建 React。
3. 校验生产 Compose。
4. SSH 登录服务器。
5. 备份 PostgreSQL，保留七天。
6. 快进拉取代码。
7. 创建共享网络（如果缺失）。
8. 重建 backend/frontend。
9. 等待两个服务健康。

## 9. 提交顺序

推荐严格按以下顺序：

1. 创建 `lebenslauf.markima.de` DNS。
2. 在 ZeitPlan 服务器目录创建 `.htpasswd` 并更新 `.env`。
3. 提交、推送 ZeitPlan 共享 Nginx 改动。
4. 扩展 Let's Encrypt 证书并 reload Nginx。
5. 在服务器克隆 CreateResume 并创建 `.env`。
6. 在 GitHub 配置 CreateResume Secrets。
7. 提交、推送 CreateResume 共享网络改动。
8. 访问 `https://lebenslauf.markima.de` 并输入 Basic Auth 用户名 `lebenslauf`。

## 10. 运维检查

CreateResume：

```bash
cd /opt/create-resume
docker compose --env-file .env -f compose.prod.yml ps
docker compose --env-file .env -f compose.prod.yml logs backend --tail 100
```

ZeitPlan 共享入口：

```bash
cd /path/to/ZeitPlan
docker compose --env-file .env -f docker-compose.prod.yml logs nginx --tail 100
docker compose --env-file .env -f docker-compose.prod.yml exec nginx nginx -t
```

网络别名检查：

```bash
docker network inspect public-proxy
```

不要执行 `docker compose down -v`，它会删除对应项目的数据库卷。
